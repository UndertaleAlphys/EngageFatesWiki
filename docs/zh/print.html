这个 Mod 做了什么？

本 Mod 旨在对《Fire Emblem Engage》的整体游戏体验进行全面重制，内容包括但不限于：

  • 对核心系统的全面重构
  • 重新设计的武器与道具
  • 深度调整的技能与纹章士
  • 重新构想的地图与关卡设计

✅ 当前状态： 本 Mod 已完成主要内容制作，现已进入稳定版本阶段。

👉 你可以通过侧边栏查看更为详细的内容说明。

Mod 安装教程

本文将介绍 结合命运 mod 的安装方法与常见问题说明。

一、运行环境要求

  • Ryujinx Emulator
    • 游戏：Fire Emblem Engage
    • 版本：v2.0.0
    • 请确保游戏本体可以在 Ryujinx 上正常启动并运行。

二、依赖组件

  • Cobalt
    • 推荐版本：v1.25.4 或更高
    • 本 Mod 基于 Cobalt 框架运行，低版本可能导致功能缺失或启动失败。

Cobalt 安装方式

  1. 下载 Cobalt 的 release 包。
  2. 解压压缩文件。
  3. 将 release 包中的所有内容直接拖入 SD 卡根目录。

无需进行额外配置。

三、文件结构与安装路径

本 Mod 由以下两个部分组成，二者缺一不可：

  1. IF mod (Cobalt)

    • Mod 本体

  2. IFmodSoundBank

    • 音乐与音效资源包

安装路径

请将以上两个文件夹完整放入以下路径： sd:/engage/mods/
├─ IF mod (Cobalt)/
└─ IFmodSoundBank/

四、常见错误与注意事项

1. 启动时报错

请依次检查：

  • Mod 是否安装在 正确的路径
  • 文件夹名称是否被修改
  • 是否遗漏 IFmodSoundBank

⚠️ 重要提示
请不要修改 IF mod (Cobalt) 文件夹内的任何内容。
插件在启动时会进行完整性检查，擅自修改可能导致 Mod 直接报错。

2. 与其他 Mod 的兼容性

  • 不建议与 修改 romfs 的 Mod 同时使用
  • 若需要使用其他插件：
    • 请新建 独立的 Mod 文件夹
    • 不要将文件混合放入 IF mod (Cobalt) 中

3. 关于实机（Nintendo Switch）运行

  • Switch 系统 不接受包含非英文字符的路径
  • 若在实机上使用：
    • 需要使用 特制的 Converter
    • 将路径转换为 仅包含英文与数字

否则系统将无法识别该 Mod。

如仍遇到问题，请再次确认安装路径与依赖版本是否正确。

五、关于压缩包格式的说明

  • 本 Mod 必须在解压后才能正常运行
  • 请不要直接将 .zip / .7z 等压缩包放入 mods 目录

尽管部分 Mod 在保持压缩包格式的情况下仍可能被加载，但 结合命运 不支持以压缩包形式运行。

请务必在安装前完整解压，并确认实际的文件夹结构如下： sd:/engage/mods/
├─ IF mod (Cobalt)/
└─ IFmodSoundBank/

计算公式

  • 命中
    技 × 1.5 + 幸运 × 0.5 + 武器命中 + 支援命中

  • 回避
    速度 × 1.5 + 幸运 × 0.5 + 武器回避 + 支援回避 + 地形回避

  • 必杀
    (技 - 4) × 0.5 + 武器必杀 + 支援必杀

  • 必杀回避
    幸运 × 0.5 + 武器必杀回避 + 支援必杀回避

  • 杖命中
    魔力 + 技 + 武器命中

  • 杖回避
    魔防 × 1.5 + 幸运 × 0.5 + 地形回避

  • 连携攻击伤害
    敌方最大 HP 的 10%

  • 连携攻击命中率
    100

  • 武器相克
    若单位在武器相克中占优势，则：

    • 命中 +20
    • 回避 +20
    • 造成伤害 +2
    • 所受伤害 -2

追击机制

  • 追击系数
    默认为单位与敌方之间的速度差。
    （部分武器可能会对该数值进行修正。）

  • 追击条件
    当追击系数 ≥ 5 时，单位可以进行追击。

结合气

  • 若单位主动发起战斗或击败敌人：
    → 结合气 +1

  • 若在同一次行动中同时满足以上两项：
    → 结合气 +2

SP 系统

  • 是否装备纹章士戒指不影响 SP 的获取

转职

职业转换系统概述

职业等级上限

  • 基础职业：等级上限 20
  • 高级职业 / 特殊职业：等级上限 40

职业技能

  • 基础职业：
    若该职业拥有职业技能，角色在升级时即可习得。
  • 高级职业 / 特殊职业：
    角色在达到 21 级 时习得该职业技能。

职业技能与继承技能池的关系

  • 当角色在某一职业中习得该职业的职业技能后：
    • 一旦转出该职业，该职业技能将进入
      可自由装备的继承技能池
    • 可像其他继承技能一样自由装备

但需要注意：

  • 当角色转回原职业时：
    • 该职业技能会重新变回不能自由装备的状态，直到下次转出该职业

该行为属于游戏机制设计的一部分，并非 Bug。

大师之证

  • 当基础职业角色达到 10 级时，可使用 大师之证。
  • 将角色转职为对应的 高级职业。
  • 转职后，角色等级将直接变为 20 级，即使原等级更低。
  • 转职过程中不会获得任何升级带来的能力成长。

平行之证

  • 转职时，角色等级保持不变。

基础职业角色：

  • 可转职为其他 基础职业 或 特殊职业。

高级职业角色：

  • 可转职为其他 高级职业 或 特殊职业。

特殊职业角色：

  • 若 等级 < 20：
    可转职为 基础职业 或 特殊职业。
  • 若 等级 ≥ 20：
    可转职为 高级职业 或 特殊职业。

其他说明

  • 角色无法转职为当前所处的同一职业。
  • 若角色在转职后已达到该职业的等级上限，
    将会立刻习得该职业的职业技能。
    • (用于防止因无法继续升级而错过技能的情况。)

幕后实现说明

本 Mod 并不仅仅包含内容与数值调整。

为了支持其整体设计目标，我们使用了一个自定义的 Skyline 插件，对《Fire Emblem Engage》的部分运行时行为进行了扩展与调整。
本页面将从技术角度介绍 具体实现了哪些内容，以及为什么要这样做，不要求读者具备相关源码背景。

1. 文件完整性与稳定性

为了在开发与测试过程中减少不可预期的行为：

  • 插件会在启动时对 Mod 文件执行 完整性校验
  • IF mod (Cobalt) 目录下的文件会与预先记录的哈希值进行比对
  • 若检测到不一致，将直接在启动阶段报错，而不是继续运行

这一机制的目的是为复杂的运行时逻辑提供一个稳定、可预测的基础环境。

2. 职业转换与技能习得逻辑

职业转换系统中的多个关键行为由插件明确处理：

  • 晋级与转职流程被统一规范
  • 等级变化被显式控制，以避免边缘情况
  • 动态等级缩放被有意禁用

职业技能的处理方式

插件为职业技能制定了一套一致的规则：

  • 当角色习得某职业的职业技能后，
    一旦转出该职业，该技能将进入可自由装备的继承技能池
  • 当角色转回原职业时：
    • 角色会立刻重新学会该职业的职业技能
  • 对于已达到等级上限、无法继续升级的情况，也进行了额外处理

这一设计旨在让技能的获得与切换行为更加直观、可理解。

3. 战斗计算流程的扩展

部分战斗计算并非仅通过数据修改完成，而是在运行时对计算流程进行了扩展：

  • 命中、回避、必杀、必杀回避等公式被调整
  • 追击条件被重新定义
  • 武器相克效果被统一处理
  • 地形与支援修正被整合到一致的判定逻辑中

这些调整的目标并非单纯改变强度，而是让战斗结果更加清晰、可预测。

4. 结合气与 SP 行为

结合气

  • 原有部分结合气累积逻辑被抑制
  • 结合气的获取改为基于以下行为进行计算：
    • 主动发起战斗
    • 击败敌人
    • 单次行动中同时满足上述条件

SP 系统

  • SP 的获取不再依赖是否装备纹章士
  • 角色获得的 SP 等于获得的 EXP

这些规则通过运行时逻辑统一执行，以保证在各种场景下的一致性。

5. 炮台系统的调整

地图炮台系统进行了多项改进：

  • 炮弹不再消耗
  • 剩余弹药显示为无限
  • 支持多目标选择
  • 不同类型炮台的射程显示方式被统一

目标选择逻辑也进行了调整，使其更贴近实际战斗影响。

6. 技能行为的修正与扩展

对部分技能行为进行了修正或补充：

  • 再移动＋＋ 改为提供剩余移动力
  • Style Bonus 变体技能在条件变化时能够正确切换与卸除
  • 地形有效性判定在伤害、回复与站位逻辑中统一
  • 武器熟练度判定被扩展以支持自定义资质规则

这些改动主要集中在行为一致性与正确性上。

7. 道具逻辑扩展

在必要的场景下，对道具行为进行了扩展：

  • 可通过附带技能标签覆盖治疗量
  • 部分祝福类道具放宽了可使用条件
  • 在受控场景中，适当放宽了道具使用限制

所有调整均基于运行时条件进行判定。

8. 脚本与事件扩展

插件为事件脚本提供了额外的功能接口：

  • 可直接操作结合气、回合数与牵绊等级
  • 提供更复杂的目标筛选逻辑（如妨害目标选择）
  • 支持自定义地图效果与条件判断
  • 提供用于特定关卡的实用函数

这些扩展使复杂机制可以通过脚本表达，而无需硬编码在关卡中。

9. 有意限制的系统

出于整体设计与节奏考虑，一些系统被有意限制或禁用：

  • 井系统
  • 投资系统
  • 遭遇战

这些调整是 Mod 整体设计的一部分。

总结

该插件作为 Mod 的技术支撑框架：

  • 对现有系统进行了有限而明确的扩展
  • 处理了一些难以通过数据层解决的边缘问题
  • 为整体机制提供了更稳定、可控的实现方式

其目标并非替代原有系统，而是在尊重原设计的基础上，
对其进行谨慎而必要的调整。
